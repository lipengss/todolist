交互流程

1. 页面加载 → 调用 GET /api/auth/captcha 获取 sliderToken
2. 用户 按住滑块 → 开始拖动 → 滑块跟随鼠标/touch 移动
3. 滑块 到达最右端 → 锁定位置 → 显示 "验证通过" + 绿色对勾
4. 用户提交登录 → sliderToken + 登录凭据一起发送
5. 后端校验：token 存在 + 未过期 + 时间差 ≥ 800ms（防止脚本秒过）

后端变更

GET /api/auth/captcha → { token }
• 后端只返回一个 UUID token，不生成图片，极简
• 内存存储 token → timestamp（5分钟过期）

POST /api/auth/login + sliderToken
• 校验：token 存在 + now - createdAt ≥ 800ms + 未过期
• 通过后删除 token，防止重用

优点

  • 零看不清问题，操作直觉
  • 后端无需生成图片，代码更少
  • 移动端友好（拖拽手势天然支持）

注意

  • 需处理 touch + mouse 双事件
  • 小于 800ms 的提交会被拒绝
