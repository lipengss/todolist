滑块验证码：两种方案

替换 SVG 字符验证码，零第三方依赖，自托管

A

拼图滑块验证

拖动拼图块到缺口位置对齐。类似阿里云/腾讯云滑块验证。

→

原理：后端发一张带缺口的背景图 + 拼图块位置 → 用户拖动滑块对齐 → 前端发回滑块最终位置 → 后端校验误差

B

简单滑条验证

拖动滑条从左到右到底即通过。检测人类行为而非视觉识别。

→
→ 拖动到最右边

原理：用户拖动到底 → 后端检测滑动轨迹（速度、加速度、停顿点）→ 判断是否为人类操作

对比

A. 拼图滑块 B. 简单滑条
安全性 高，需匹配位置 中，靠行为检测
实现复杂度 中，需图片处理 低，纯前端交互
后端依赖 需要 sharp/canvas 生成图片 仅需记录滑动 token
用户体验 "对齐拼图"直观 "滑到底"简单
